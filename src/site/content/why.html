A Sinatra inspired micro web framework for quickly creating web applications in Java with minimal effort

Why?

Spark's intention is not to compete with Sinatra, or the dozen of clones in different languages (Grafitti, Nancy, Flask, Sammy etc...). Its intention is to be used by Java developers that want or are required to develop in pure Java.

Spark focuses on being as simple and straight-forward as possible, without the need for cumbersome (XML) configuration, to enable very fast web application development in pure Java with minimal effort.
